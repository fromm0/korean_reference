Local 애플리케이션 활용

VisualVM을 실행하면, Applications창의 Local노드아래 각각의 로컬 자바 애플리케이션의 노드가 표기된다. 애플리케이션의 프로파일러 스냅샷, 쓰레드 덤프 그리고 힙 덤프를 보기 위해 애플리케이션을 열어볼수도 있다. 애플리케이션 노드나 하위 노드에서 마우스 우측 버튼을 클릭하는 것으로 다양한 기능을 처리할 수 있는 팝업 메뉴를 띄울수 있다.
알림: 로컬 애플리케이션의 목록은 VisualVM에서 항상 노출된다.

로컬 애플리케이션 노드는 팝업메뉴에서 몇가지 기능은 항상 제공한다.

  • Open. 애플리케이션 데이터를 볼수 있도록 메인창에 애플리케이션을 위한 탭을 열기
  • Thread Dump. 애플리케이션의 쓰레드 덤프를 생성
  • Heap Dump. 애플리케이션의 힙 덤프를 생성
  • Profile. 애프리케이션을 프로파일링할 수 있도록 프로파일러 하위탭을 열기 (주의할 점은 애플리케이션 목록에서 VisualVM이 보이더라도 VisualVM자체를 프로파일링할 수는 없다.)
  • Profiler Snapshot. 프로파일링 세션에서 결과에 대한 스냅샷을 생성 스냅샷을 메인창에서 열면 애플리케이션 노드 아래 스냅샷을 위한 노드가 표기된다. 이 기능은 VisualVM이 애플리케이션을 프로파일링할때만 사용가능하게 된다.
  • Application Snapshot. 애플리케이션 스냅샷을 생성하는 것은 쓰레드및 힙과 같은 모든 애플리케이션 덤프와 스냅샷을 생성 애플리케이션 스냅샷을 위한 노드는 Snapshots노드 아래 표기된다.
  • Enable Heap Dump on OOME. 애플리케이션에 OutOfMemoryException이 발생했을때 자동으로 힙 덤프를 생성해야 한다면 이 기능을 사용

이제부터 설명하는 각각의 기능은 팝업메뉴에서 직접 실행할 수 있다.

애플리케이션 데이터 보기

VisualVM은 메인창에서 실행중인 각각의 로컬 애플리케이션에 대해서 각각의 탭을 열어서 데이터를 보여준다. 각각의 애플리케이션 탭은 필요한 정보를 보여주기 위해 하위탭을 가지기도 한다.

애플리케이션 탭을 열기위해, Applications창의 애플리케이션 노드에서 마우스 우측 버튼을 클릭하고 Open를 선택한다. (또는 애플리케이션 노드를 더블클릭해도 된다.) Open메뉴를 선택하면, 메인창에 애플리케이션 탭에서 개요에 해당하는 Overview탭이 열린다.

로컬 애플리케이션 탭은 4개의 하위 탭을 가진다.

  • Overview. 좀더 많은 것을 보고싶다면 애플리케이션 개요보기를 보라
  • Monitor. 좀더 많은 것을 보고싶다면 애플리케이션 모니터링를 보라
  • Threads. 좀더 많은 것을 보고싶다면 애플리케이션 쓰레드 모니터링를 보라
  • Profiler. 좀더 많은 것을 보고싶다면 애플리케이션 프로파일링를 보라

추가적인 탭과 하위 탭은 설치된 플러그인의 종류에 따라 갯수가 다를수 있다.

쓰레드 덤프 생성

쓰레드 덤프를 생성한 시점의 실제 애플리케이션 쓰레드에 대한 정보를 보기 위해 VisualVM은 쓰레드 덤프 생성 기능을 제공한다. 쓰레드 덤프를 생성하면, 쓰레드 덤프는 메인창의 애플리케이션 하위 탭으로 열린다. 쓰레드 덤프를 위한 노드는 Applications창에서 애플리케이션 노드 아래 나타난다.

아래에서 설명하는 방법으로 쓰레드 덤프를 생성할 수 있다.

  • Applications창에서 애플리케이션 노드를 선택하고 마우스 우측 버튼을 클릭한 뒤 나오는 팝업 메뉴에서 Thread Dump를 선택한다.
  • 애플리케이션 탭에서 Threads탭에 있는 Thread Dump 버튼을 클릭한다.

쓰레드 덤프를 생성하면, 쓰레드 덤프는 메인창의 애플리케이션에서 하위 탭으로 열린다. 쓰레드 덤프를 위한 노드는 애플리케이션 노드아래 표시된다.

애플리케이션 쓰레드를 활용하는 방법에 대해서는 다음 문서를 참고하자.

힙 덤프 생성

힙 메모리내에 존재하는 객체의 스냅샷을 보기 위해 로컬 애플리케이션의 힙 덤프를 생성할 수 있다.

아래에서 설명하는 방법으로 힙 덤프를 생성할 수 있다.

  • Applications창에서 애플리케이션 노드를 선택하고 마우스 우측 버튼을 클릭한 뒤 나오는 팝업 메뉴에서 Heap Dump를 선택한다.
  • 애플리케이션 탭에서 Monitor탭에 있는 Heap Dump 버튼을 클릭한다.

힙 덤프를 생성하면, 힙 덤프는 메인창의 애플리케이션에서 하위 탭으로 열린다. 힙 덤프를 위한 노드는 애플리케이션 노드아래 표시된다.

힙 덤프를 활용하는 방법에 대해서는 다음 문서를 참고하자.

  • 힙덤프 살펴보기

애플리케이션 프로파일링

VisualVM은 로컬 애플리케이션의 성능과 메모리 사용내용을 분석할 수 있도록 프로파일러를 포함하고 있다. VisualVM이나 로컬 애플리케이션을 재시작할 필요없이 애플리케이션을 프로파일링할 수 있다. 프로파일링을 시작하기 위해서는, 프로파일링 탭을 열기 위해 애플리케이션 노드를 선택하고 마우스 우측 버튼을 클릭해서 나오는 팝업메뉴에서 Profile을 선택한다.
알림: VisualVM은 VisualVM이 아닌 다른 자바 애플리케이션만 프로파일링할 수 있다.

프로파일링에 대해 좀더 알고 싶다면 다음 문서를 참고하자.

스냅샷 생성

VisualVM은 스냅샷을 생성한 시점의 애플리케이션 데이터를 볼 수 있도록 스냅샷 생성이 가능하다. 로컬 시스템에 스냅샷 파일을 저장하고 나중에 다시 보거나 다른 사람에게 줄수도 있다.

VisualVM은 두가지의 스냅샷 타입을 제공한다.

  • 프로파일러 스냅샷 프로파일러 스냅샷은 스냅샷을 생성하는 시점의 프로파일링 데이터를 가진다. 스냅샷은 메모리 스냅샷과 CPU스냅샷으로 구성된다. 프로파일러 스냅샷은 애플리케이션을 프로파일링할때만 생성이 가능하다.
  • 애플리케이션 스냅샷 애플리케이션 스냅샷은 힙덤프, 쓰레드덤프 그리고 애플리케이션 노드아래에 나열되는 스냅샷 모두를 생성한다.

스냅샷에 대해 좀더 알고 싶다면 다음 문서를 참고하자.
